arabic cheque ocr pipeline

every digit. every word. verified.

a unified pipeline that detects cheque fields, reads courtesy digits and handwritten Arabic legal amounts, then cross-validates them. precision-first. deployed serverlessly.

open demo → github

a master's project · KFUPM

why

handwritten cheques carry billions every year. machines that read them must be precise — not just with digits, but with every word of the legal amount. we built a system that reads both, then cross-checks. because trust requires verification.

pipeline

four stages. one container. full verification.

01

detect

cascade r-cnn with resnet-50 + fpn localizes exactly one courtesy box and one legal box per cheque. trained on annotated Arabic bank cheques.

02

read digits

crnn + bidirectional lstm with ctc decoding transcribes the courtesy amount crop. arabic-indic numerals (١٢٣) normalized to standard digits (123). 96.65% digit accuracy.

03

read words

qwen3.5-0.8b fine-tuned with lora reads the handwritten Arabic legal amount. a rule-based parser with edit-distance recovery converts the text to a number.

04

verify

the parsed legal amount is cross-checked against courtesy digits. on mismatch, courtesy-guided image enhancement fallback retries with border-cleaned and padded variants.

features

built for real cheques.

dual ocr engines

crnn+ctc for digit sequences, qwen3.5 vision-language model for handwritten Arabic text. each model optimized for its domain.

courtesy-guided fallback

when the legal transcription doesn't match, the system re-processes with enhanced crops — border cleanup, auto-contrast, white padding — guided by the courtesy digits.

arabic-to-number parser

custom rule-based parser handles split connectors, merged words, and levenshtein edit-distance corrections for unrecognized tokens.

cross-verification

the final output includes a verification verdict — match or mismatch — with full transparency on both parsed amounts.

serverless gpu

deployed on modal with an a10g gpu. cold start in seconds. scales to zero when idle. no infrastructure to manage.

under the hood

detector

courtesy ocr

crnn + bilstm + ctc · custom pytorch architecture

legal ocr

qwen3.5-0.8b · lora fine-tuned · peft · transformers

preprocessing

opencv · pillow · clahe · morphological background subtraction

interface

streamlit · base64 inline rendering for serverless compatibility

deployment

modal · a10g gpu · persistent volumes · serverless web endpoint
